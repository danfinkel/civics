Learned — image format
PDFs saturate the visual token budget
High-DPI PDF renders consume the model's attention on layout elements before reaching document text. Phone photos at 100 DPI outperform PDF scans — 100% deadline accuracy vs 0%.
Built — capture guidance
App routes residents to photograph, not scan
All inference runs on JPEG input only. Image pre-processing downscales to 2048px max before OCR — enough for text, too small for layout token saturation.
_maxDimension = 2048 px
Learned — prompt design
Generic field names triggered hallucination
Prompts using key_date, document_type produced 33% hallucination on JPEG inputs and wholesale template substitution on PDFs. The model had no basis for schema-to-document mapping.
Built — semantic prompts
Per-document prompts aligned to DTA notice schema
Field names match the document's information architecture: response_deadline, requested_category, consequence. Hallucination eliminated entirely on JPEG inputs. Effect size exceeded model scaling.
0% hallucination · semantic prompts · JPEG inputs

Learned — image quality
Blur tolerant — but rotation, crop, and zoom are real failures
Deadline extraction held at 100% across all blur levels. But rotation (~35°), documents too small in frame, and cropped fields caused complete extraction failure. Laplacian variance detected none of them.
Built — four-rule detector
Pre-processing gate calibrated on real iPhone photos
Four rules replace Laplacian variance: scene entropy, rotation angle, frame aspect ratio, white region coverage. Each rule maps to an actionable resident instruction. 72% accuracy across 100 test cases.
entropy · rotation · aspect · coverage
Learned — model constraints
4,096-token context requires deliberate budget design
On-device inference with a 2B GGUF model has hard context limits. Unmanaged, full-resolution OCR text from a single government notice overflows the prompt budget and produces silent truncation.
Built — four-layer token budget
Image → OCR → prompt → output, each capped and tested
Image downscaled to 2048px. OCR capped at 3,500 chars per notice, 5,800 total. Prompt hard-clamped at 8,000 chars. Output budget 2,048 tokens — 4× realistic max, safety net against repetition loops.
noticeMax=3500 · totalMax=5800 · promptMax=8000
